Notes

  • selecting the element form will return a collection not a single form (even if there is just one form at the document)

    the attribute name helps to retrive the distinct form

    Form events:

    • focus: when the element is focused on
    • blur: when the user moves the focus away
    • change: after losing focus AND the content is changed
    • submit: send the form data to its destiny

    Custom validation: when binding an event to a submit form, we could add a callback that will increase the custom validation required.

    the attribute noValidate could be used when we want to handle manually errors

    good source to learn about form constraints validation AND

    types of client-side validation: built-in and javascript
